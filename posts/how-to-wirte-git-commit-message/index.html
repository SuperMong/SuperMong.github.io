How to write a commit message in Git?

November 9, 2021 · SuperMong

Since I used git, I have always wanted to know how to write a concise and clear commit message? After entering the job, I found that Angular’s specification is the most popular specification. So, let’s start learning it.

Commit Message Format#

Each commit message consists of a header, a body and a footer:

<header>
<BLANK LINE>
<body>
<BLANK LINE>
<footer>

Header#

A commit message must has header, and it has its own format:

<type>(<scope>): <short summary>

The type field is mandatory, must be one of following:

  • build: Changes that affect the build system or external dependencies
  • ci: Changes to our CI configuration files and scripts
  • docs: Documentation only changes
  • feat: A new feature
  • fix: A bug fix
  • pref: A code change that improves performance
  • refactor: A code change that neither fixes a bug nor adds a feature
  • test: Adding missing tests or correcting existing tests

The scope field is optional, should be the name of the package affected.

And summary is the description of the commit. It’s also mandatory and there are some rules to write it:

  • use imperative
  • don’t capitalize the first letter
  • no dot (.) at the end

Body#

If commit type is docs, then body is required, In other cases, it’s optional.

Just like summary, explain why you are making the change.

Footer#

The footer can contain information for breaking changes and deprecations, and is also a place to reference issues.
